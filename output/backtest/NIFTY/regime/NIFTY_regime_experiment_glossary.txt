NIFTY regime experiment glossary

Files:
- NIFTY_regime_experiment_comparison.csv: date-level feature, hindsight, regime, and match details.
- NIFTY_regime_experiment_summary.csv: experiment-level metrics.

Core comparison columns:
- date: Trading date being evaluated.
- hindsight_regime: Future-looking diagnostic regime label. This is not available at trade time.
- hindsight_regime_minus_1d: Hindsight regime from the prior trading row.
- hindsight_regime_minus_2d: Hindsight regime from two trading rows earlier.
- lagged_hindsight_trend_2d: Trend reference label for date D. It is TREND_UP or TREND_DOWN if hindsight_regime on D, D-1, or D-2 is a trend; otherwise blank.
- {experiment}_regime: Regime detected by that experiment on date D.
- {experiment}_lag2_trend_match: True when {experiment}_regime matches lagged_hindsight_trend_2d.
- {experiment}_buffer_match: True when {experiment}_regime matches same-day hindsight_regime, or recent hindsight within the configured buffer window.

Summary columns:
- experiment: Name of the regime rule variant.
- buffer_days: Prior trading rows allowed for the broad buffered match.
- comparable_rows: Rows where same-day hindsight_regime is available and not UNKNOWN.
- overall_buffered_correct: Count of comparable rows where the experiment matched same-day/recent hindsight.
- overall_buffered_accuracy_pct: overall_buffered_correct / comparable_rows.
- lag2_trend_rows: Rows where lagged_hindsight_trend_2d is TREND_UP or TREND_DOWN.
- trend_signal_rows: Rows where the experiment detected TREND_UP or TREND_DOWN.
- lag2_trend_match_count: Count where the experiment's trend signal matched lagged_hindsight_trend_2d. This is the shared numerator for recall and precision.
- lag2_trend_recall_pct: lag2_trend_match_count / lag2_trend_rows. Main trend-detection recall metric.
- trend_signal_lag2_precision_pct: lag2_trend_match_count / trend_signal_rows. Main trend-signal precision metric.
- trend_up_buffered_recall_pct: Among same-day hindsight TREND_UP rows, percent caught by broad buffered match.
- trend_down_buffered_recall_pct: Among same-day hindsight TREND_DOWN rows, percent caught by broad buffered match.
- range_buffered_recall_pct: Among same-day hindsight RANGE rows, percent caught by broad buffered match.
- choppy_buffered_recall_pct: Among same-day hindsight CHOPPY rows, percent caught by broad buffered match.

How to read the trend metrics:
- For trend-regime experiments, prioritize lag2_trend_recall_pct and trend_signal_lag2_precision_pct.
- lag2_trend_recall_pct answers: "When hindsight trend happened on D, D-1, or D-2, did this experiment confirm that trend on D?"
- trend_signal_lag2_precision_pct answers: "When this experiment called trend, how often was that trend aligned with recent hindsight trend?"
- overall_buffered_accuracy_pct is useful context, but it can favor conservative RANGE-heavy detectors and should not be the main metric when tuning TREND_UP/TREND_DOWN detection.

Experiment definitions:
- current: Production regime detector.
- current_trend_ret60_015: Same as current trend rule, but ret_60d threshold is 1.5% instead of 2%.
- current_trend_ret60_010: Same as current trend rule, but ret_60d threshold is 1.0% instead of 2%.
- current_trend_eff60_020: Same as current trend rule, but trend_efficiency_60d threshold is 0.20 instead of 0.25.
- current_trend_ma50_flat_ok: Same as current trend rule, but ma50_slope may be flat at 0 instead of strictly positive/negative.
- current_trend_ma20_slope_only: Same as current trend rule, but only ma20_slope is required; ma50_slope is ignored.
- current_trend_stack_ma10_ma20: Uses close > ma10 > ma20 for TREND_UP and close < ma10 < ma20 for TREND_DOWN, with ma10d_slope and ma20_slope alignment.
- current_trend_ret20_eff20: Same MA stack/slope as current, but uses ret_20d >= 1.5% and trend_efficiency_20d >= 0.30 for TREND_UP; reversed for TREND_DOWN.
- current_trend_ret10_eff10: Uses close > ma10 > ma20, ma10d_slope > 0, ma20_slope > 0, ret_10d >= 1.0%, and trend_efficiency_10d >= 0.35 for TREND_UP; reversed for TREND_DOWN.
- current_trend_range10_ret5_eff5: Uses 10-day range position plus 5-day return/efficiency. TREND_UP when range_position_10d >= 0.65, ret_5d >= 0, and trend_efficiency_5d >= 0.35. TREND_DOWN when range_position_10d <= 0.30, ret_5d <= 0, and trend_efficiency_5d >= 0.35.
- current_trend_range20_ret5_ma5: Uses 20-day range position plus 5-day return and ma5d_slope. TREND_UP when range_position_20d >= 0.65, ret_5d >= 0, and ma5d_slope >= 0. TREND_DOWN when range_position_20d <= 0.30, ret_5d <= 0, and ma5d_slope <= 0.
- current_trend_ret5_eff5_ma5: Uses fast 5-day momentum. TREND_UP when ret_5d >= 0.75%, trend_efficiency_5d >= 0.45, and ma5d_slope > 0. TREND_DOWN when ret_5d <= -0.75%, trend_efficiency_5d >= 0.45, and ma5d_slope < 0.
- current_trend_ret10_range_eff10: Uses 10-day return, range position, and efficiency. TREND_UP when ret_10d >= 1.0%, range_position_10d >= 0.70, and trend_efficiency_10d >= 0.30. TREND_DOWN when ret_10d <= -1.0%, range_position_10d <= 0.30, and trend_efficiency_10d >= 0.30.
