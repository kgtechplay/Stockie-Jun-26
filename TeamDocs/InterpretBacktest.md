# Interpreting Backtest Results

Reference for every column produced by the two backtest scripts.

---

## NIFTY_prediction.csv

Generated by `python backtest/test_underlying_prediction.py`.  
One row per trading day with a `SignalFeatureDaily` entry.  
A companion `NIFTY_prediction_summary.txt` is also written with accuracy and recall stats per strategy.

### Identity

| Column | What it means |
|---|---|
| `date` | Trading date for this row. |
| `underlying` | Underlying symbol — always `NIFTY` for this file. |

### Raw features (from SignalFeatureDaily)

These are pre-computed technical features stored in the DB. `NaN` for early rows where there was insufficient history to compute the indicator.

| Column | What it means |
|---|---|
| `close` | Closing price at 15:15 (market close proxy). |
| `ma10` | 10-day simple moving average of close. |
| `ma20` | 20-day simple moving average of close. |
| `ma50` | 50-day simple moving average of close. |
| `ma90` | 90-day simple moving average of close. |
| `rsi14` | 14-period RSI (0–100); above 65 = overbought, below 35 = oversold. |
| `atr14` | 14-period Average True Range in index points — measures daily swing magnitude. |
| `bb_upper` | Upper Bollinger Band (MA20 + 2σ). |
| `bb_lower` | Lower Bollinger Band (MA20 − 2σ). |
| `bb_width` | Band width = (upper − lower) / middle; higher = more volatile / expanding. |
| `ret_5d` | Price return over the past 5 trading days (decimal, e.g. 0.02 = +2%). |
| `ret_20d` | Price return over the past 20 trading days. |
| `ret_60d` | Price return over the past 60 trading days. |
| `volatility_20d` | 20-day realized volatility — annualised standard deviation of daily returns. |
| `regime` | Market regime detected from features: `TREND_UP`, `TREND_DOWN`, `RANGE`, `CHOPPY`, or `UNKNOWN`. |

Note: `volume_ratio` uses NIFTY near-month futures volume sourced from NSE FO bhavcopy (not the index itself, which has no traded volume). Populated by `scripts/backfill_NIFTY/backfill_nifty_volume.py`.

### Per-strategy raw signals

Each column is the signal from one strategy. Values: `CALL` / `PUT` / `NO_POSITION`.

Only the strategies relevant to the day's regime are executed; all others are recorded as `NO_POSITION` without running.

| Regime | Active strategies |
|---|---|
| `TREND_UP` | `MaTrend_001`, `trendUpRangeBreakout`, `BollingerMeanReversion` |
| `TREND_DOWN` | `MaTrend_001`, `trendDownRangeBreakout`, `BollingerMeanReversion` |
| `RANGE` | `rangeBollingerMeanReversion`, `rangeRsiMeanReversion_6535` |
| `CHOPPY` / `UNKNOWN` | (none — no signal possible) |

| Column | Strategy logic |
|---|---|
| `BollingerMeanReversion` | CALL when close < lower BB; PUT when close > upper BB. Active in `TREND_UP` / `TREND_DOWN` as confirmation. |
| `MaTrend_001` | CALL when MA10 > MA20 by ≥0.1%; PUT when inverted. Active in trend regimes. |
| `rangeBollingerMeanReversion` | Same BB logic but only active in `RANGE` regime — fades extremes within the band. |
| `rangeRsiMeanReversion_6535` | CALL when RSI < 40; PUT when RSI > 60. Active in `RANGE` regime only. (Thresholds relaxed from 35/65 in pipeline to fire in moderate RANGE conditions.) |
| `trendUpRangeBreakout` | CALL when close breaks above the 20-day high. Active in `TREND_UP` only. |
| `trendDownRangeBreakout` | PUT when close breaks below the 20-day low. Active in `TREND_DOWN` only. |
| `RsiMeanReversion_6535` | Ungated RSI MR — currently inactive (superseded by the range-gated version). |
| `choppy` / `unknown` | Always `NO_POSITION` — placeholder columns for the veto regimes. |

### Aggregated prediction

Produced by combining the nine strategy signals through `aggregator.py`, `scoring.py`, and `view.py`.

| Column | What it means |
|---|---|
| `raw_signal` | Consensus across all strategies: `CALL`, `PUT`, or `NO_POSITION`. Conflicting signals → `NO_POSITION`. |
| `direction` | Human-readable direction: `BULLISH`, `BEARISH`, or `NEUTRAL`. |
| `strength_score` | 0–100 composite score; sum of the five score components below. ≥65 qualifies for option selection. |
| `confidence` | Bucketed strength: `HIGH` (≥80), `MEDIUM` (≥65), or `LOW` (<65). |
| `option_bias` | `BULLISH_STRONG`, `BULLISH_MODERATE`, `BEARISH_MODERATE`, `BEARISH_STRONG`, or `NEUTRAL`. Drives option strategy type selection. |
| `is_option_eligible` | `True` if `raw_signal` is not `NO_POSITION`, `strength_score` ≥ 65, and regime is not `CHOPPY`. Only `True` rows proceed to option selection. RANGE signals typically score 30–50 so `is_option_eligible` will be `False`; they are backtest-only signals. |

### Score components

These three values contribute to `strength_score`. They show *why* a score is high or low.

The following are not included for NIFTY:
- `sector/benchmark_confirmation_score` — always 0 (no sector/benchmark regime in pipeline)
- `relative_strength_score` — sector/benchmark RS always `None`; `ret_20d` (the only remaining contributor) is already a raw feature column

| Column | What it contributes |
|---|---|
| `stock_technical_score` | Points from MA alignment, RSI position, and Bollinger band signals (max 25). |
| `volume_confirmation_score` | Points from NIFTY futures volume vs 20-day average — 10 pts if ratio ≥1.5, 7 if ≥1.1, 4 if ≥0.8. 0 until a signal fires (scoring only runs on active signals). |
| `risk_quality_score` | Points for a favourable risk profile — lower ATR and volatility score better (max 10). |
| `regime_quality_score` | Points for a clear, strong regime — `TREND_UP`/`TREND_DOWN` scores up to 15; `CHOPPY` subtracts 15; `RANGE` with matching setup type scores 10. |

### Backtest reference

| Column | What it means |
|---|---|
| `next_day_close_chg_pct` | NIFTY spot close change to the next trading day (%). Positive = rose; negative = fell. Accuracy benchmark (compare to `raw_signal` direction) and recall benchmark (|value| ≥ 0.5% counts as a significant move). |

## NIFTY_optionSelection.csv

Generated by `python backtest/test_optionselection_e2e.py`.  
Extends every row from `NIFTY_prediction.csv` with option selection and P&L columns.

### Option selection

Produced by `select_option_strategy()` using the EOD option chain at 15:15 on the signal date.

| Column | What it means |
|---|---|
| `selected_strategy` | Option strategy chosen: `LONG_CALL`, `LONG_PUT`, `BULL_CALL_SPREAD`, `BEAR_PUT_SPREAD`, or `NO_TRADE`. |
| `option_bias_selected` | `option_bias` value that drove strategy type selection. |
| `no_trade_reason` | Reason for `NO_TRADE` if applicable, e.g. "Underlying signal score below threshold" or "No qualifying candidates". |
| `evaluated_candidate_count` | Number of option candidates scored before choosing the best. |
| `strategy_direction` | `BULLISH` or `BEARISH` as determined by the option selector. |
| `entry_debit_or_credit` | Net premium paid (debit > 0) or received (credit < 0) to enter the strategy in points. |
| `max_profit` | Maximum possible profit at expiry in points (unbounded for naked long options). |
| `max_loss` | Maximum possible loss in points (= premium paid for long strategies). |
| `breakeven` | Underlying price at which the strategy breaks even at expiry. |
| `reward_risk` | `max_profit / max_loss`; higher is better. |
| `selection_score` | 0–100 candidate score from `scoring.py`; the highest-scoring candidate is selected. |
| `selection_confidence` | `HIGH`, `MEDIUM`, or `LOW` confidence in the selected candidate. |
| `total_delta` | Net delta of the position — directional exposure to a 1-point move in NIFTY. |
| `total_theta` | Net theta — daily time decay in points (negative = costs money per day held). |
| `total_vega` | Net vega — sensitivity to a 1% change in implied volatility. |
| `legs_summary` | Human-readable summary of each leg, e.g. `BUY NIFTY26JUN23800CE @145.5`. |
| `selection_reasons` | Pipe-separated reasons the candidate was chosen. |
| `selection_warnings` | Pipe-separated warnings (e.g. wide spread, low OI). |

### Primary BUY leg details

The contract used for P&L tracking — the bought leg of the strategy.

| Column | What it means |
|---|---|
| `primary_buy_token` | Kite `instrument_token` for the bought leg. |
| `primary_buy_symbol` | Kite trading symbol, e.g. `NIFTY26JUN23800CE`. |
| `primary_buy_strike` | Strike price of the bought option. |
| `primary_buy_expiry` | Expiry date of the bought option. |
| `primary_buy_option_type` | `CE` (call) or `PE` (put). |
| `primary_buy_entry_price_signal` | `last_price` of the option at the time of signal (15:15 on signal date) — for reference only, not the actual entry. |
| `primary_buy_iv` | Implied volatility of the bought leg at signal time. |
| `primary_buy_delta` | Delta of the bought leg at signal time. |

### P&L backtest

Entry and exit are based on `OptionSnapshot.last_price` rows for the bought leg on the next trading day.

| Column | What it means |
|---|---|
| `next_trading_day` | The trading day after the signal date — when the trade is entered. |
| `entry_time` | Timestamp of the first snapshot on `next_trading_day` — used as the entry price reference. |
| `entry_price` | Option price at `entry_time` (first snapshot of the day = open proxy). |
| `exit_time` | Timestamp of the last snapshot on `next_trading_day` — EOD close proxy. |
| `exit_price` | Option price at `exit_time`. |
| `lot_size` | NIFTY lot size at the time (used for `pnl_per_lot`). |
| `pnl_per_unit` | `exit_price − entry_price` in points per unit. |
| `pnl_per_lot` | `pnl_per_unit × lot_size` — actual rupee P&L for one lot. |
| `return_pct` | `pnl_per_unit / entry_price × 100` — percentage return on the option premium. |
| `option_result` | `PROFIT`, `LOSS`, or `BREAKEVEN` based on `pnl_per_unit`. |
| `first_2pct_profit_datetime` | First snapshot timestamp (across the next 5 trading days) at which the option price reached ≥ entry × 1.02. `null` if the 2% target was never hit in that window. |
